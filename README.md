# Preventive Maintenance Checklist System / PM Archive

## Project Description

The **Preventive Maintenance Checklist System**, also known as **PM Archive**, is a web-based system for organizing and managing preventive maintenance records.

It helps administrators and staff keep track of maintenance forms, checklist data, equipment or item conditions, user reports, offices or colleges, departments, and archived or backup records in one database-driven application.

The system is built with:

- Laravel 10
- Vue 3
- Vite
- MySQL
- XAMPP

It is intended for local or internal deployment, such as inside the XAMPP `htdocs` folder using a project folder like `/pm-archive`.

## Main Features

- User login and authentication
- Role-based user access
- Manage preventive maintenance forms and records
- Create, edit, view, print, export, and archive records
- Manage checklist items used in maintenance forms
- Manage offices, colleges, and departments
- Database-driven dropdowns for office/college and department selection
- Generate preventive maintenance reports
- Include user reports data inside generated reports
- Print or export records and reports, depending on enabled modules
- Archive or backup system support
- Activity logs, if enabled in the system

## User Roles

The system may support different role-based users depending on the configured database seeders and user management module.

After running the database seeders, the following default users are available:

| Role | Username / Email | Password |
| --- | --- | --- |
| Super Admin | `superadmin@example.com` | `Password123!` |
| Admin | `admin@example.com` | `Password123!` |
| Encoder | `encoder@example.com` | `Password123!` |

For local or internal deployment, change these default passwords after first login.

### Admin

Admins usually have full access to the system. They can manage users, offices or colleges, departments, checklist items, preventive maintenance records, reports, archives, and system settings.

### Staff/User

Staff or regular users can usually create, update, and view preventive maintenance records. They may also fill out checklist forms, submit user reports, print records, and generate reports depending on their assigned permissions.

### Viewer/Operator

Viewers or operators usually have limited access. They may be allowed to view records, check reports, or print documents, but may not be allowed to delete, archive, or manage system settings.

> Exact role permissions depend on how the application roles and middleware are configured.

## How to Use the System

1. Open the system in a browser.
   - Example using XAMPP subfolder:

     ```text
     http://localhost/pm-archive/public
     ```

   - Example using Laravel development server:

     ```text
     http://127.0.0.1:8000
     ```

2. Log in using an authorized account.
   - Use an account created by the administrator or generated by the database seeders.

3. Go to the dashboard.
   - The dashboard gives access to the main system modules, such as records, reports, users, offices or colleges, departments, and archives.

4. Add or manage users.
   - Admin users can create accounts and assign roles or access levels.

5. Add offices/colleges and departments.
   - These records are used in dropdown fields when creating preventive maintenance forms and reports.

6. Create a preventive maintenance record.
   - Open the preventive maintenance form or records module.
   - Add the required information, such as office/college, department, date, equipment or item details, and assigned personnel.

7. Fill out checklist items.
   - Select or answer each checklist item based on the inspection result.

8. Select the item status or condition.
   - Common status values may include:
     - Active
     - For Repair
     - Replaced
     - Condemned
     - N/A

9. Save the record.
   - After saving, the record becomes available for viewing, editing, reporting, printing, exporting, or archiving depending on user permissions.

10. View, edit, print, export, or archive records.
    - Use the available action buttons in the records table or record details page.

11. Generate reports.
    - Open the reports module and apply the needed filters, such as office/college, department, date range, status, or user report data.

## Reports Module

The reports page is used to review and summarize preventive maintenance data.

It should include:

- Preventive maintenance records
- Checklist summaries
- Office/college filtering
- Department filtering
- User reports data
- Status or condition summaries, if available
- Export and print options, if available

Reports are useful for tracking completed maintenance activities, identifying equipment that needs repair or replacement, and preparing printed or exported documentation.

## Installation Guide

### Prerequisites

Make sure the following are installed:

- XAMPP with Apache and MySQL
- PHP 8.1 or higher
- Composer
- Node.js and npm
- MySQL database server

### Setup Steps

1. Place the project inside your web directory.

   Example:

   ```text
   C:\xampp\htdocs\pm-archive
   ```

2. Open a terminal in the project folder.

   ```bash
   cd C:\xampp\htdocs\pm-archive
   ```

3. Install PHP dependencies.

   ```bash
   composer install
   ```

4. Install JavaScript dependencies.

   ```bash
   npm install
   ```

5. Create the `.env` file.

   On macOS/Linux/Git Bash:

   ```bash
   cp .env.example .env
   ```

   On Windows Command Prompt:

   ```bat
   copy .env.example .env
   ```

6. Generate the Laravel application key.

   ```bash
   php artisan key:generate
   ```

7. Create a MySQL database.

   You may use either database name depending on your setup:

   ```text
   pm_archive_db
   ```

   or:

   ```text
   prevmaincheckdb
   ```

8. Configure the database connection in `.env`.

9. Run migrations and seeders.

   ```bash
   php artisan migrate --seed
   ```

10. Build frontend assets.

    ```bash
    npm run build
    ```

11. Open the system in the browser.

    ```text
    http://localhost/pm-archive/public
    ```

    Or run Laravel locally:

    ```bash
    php artisan serve
    ```

    Then open:

    ```text
    http://127.0.0.1:8000
    ```

## XAMPP/Subfolder Setup

If the project is placed in:

```text
C:\xampp\htdocs\pm-archive
```

The browser URL may be:

```text
http://localhost/pm-archive/public
```

You may also configure Apache virtual hosts or rewrite rules so the project can be accessed through a cleaner URL, such as:

```text
http://pm-archive.test
```

When running the project in a subfolder, make sure the `.env` file uses the correct application URL.

Example:

```env
APP_URL=http://localhost/pm-archive/public
ASSET_URL=http://localhost/pm-archive/public
```

If you use a virtual host, update these values to match the virtual host URL.

## Environment Variables Example

Example `.env` database setup:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=pm_archive_db
DB_USERNAME=root
DB_PASSWORD=
```

Alternative database name:

```env
DB_DATABASE=prevmaincheckdb
```

After changing `.env`, clear the Laravel configuration cache:

```bash
php artisan config:clear
```

## Troubleshooting

### Page has no styles or JavaScript is not working

Run:

```bash
npm run build
```

For development, also run:

```bash
npm run dev
```

### Database connection error

Check the database settings in `.env`.

Confirm that:

- MySQL is running in XAMPP
- The database exists
- `DB_DATABASE` is correct
- `DB_USERNAME` and `DB_PASSWORD` are correct

Then run:

```bash
php artisan config:clear
```

### Route not defined

Check the route name or path in:

```text
routes/web.php
```

Then clear route cache:

```bash
php artisan route:clear
```

### View not found

Check that the Blade file exists in:

```text
resources/views
```

Also confirm that the controller returns the correct view name.

### Migration error

Make sure the database exists and the `.env` database settings are correct.

Then run:

```bash
php artisan migrate
```

If you need to rebuild the local database from scratch, use:

```bash
php artisan migrate:fresh --seed
```

Warning: `migrate:fresh` deletes existing database tables before recreating them.

### Config changes are not updating

Run:

```bash
php artisan config:clear
php artisan cache:clear
```

### Public or uploaded files are not accessible

If the project uses uploaded files through Laravel storage, run:

```bash
php artisan storage:link
```
